SNaPS

Role: Designer & developer (with Nathan Ho)

VST3 / AU

2024–present Sampler Plugin

Most samplers assume every note begins from scratch. But harmony doesn't work that way. If a chord changes while a violin is sustaining, the violin doesn't restart — it keeps singing through the change.

I wanted a sampler that behaved the same way. And I wanted it to work with recordings, not synthesized notes. Samples carry texture and memory in ways oscillators don't.

Origin

SNaPS began as a SuperCollider instrument that Nathan Ho and I developed over several years. Nathan built the entire engine; I kept pushing the interaction and product. For years, the SuperCollider version was my primary live instrument. The original repo is still the foundation of my performances. The plugin version is my C++ port of everything we built together.

Tagging

For years, the system was blocked by a bottleneck: every sample had to be manually tagged with its pitch class. I had thousands of samples and no time to listen to each one.

Then Nathan called me about Spotify's Basic Pitch — an open-source ML model that detects pitch in audio. I built a librarian tool around it that auto-tags entire sample libraries. The interface couldn't scale until metadata became automatic.

From SuperCollider to plugin

The plugin version (VST3/AU) packages everything we learned for DAW workflows: three playback engines — Emitter, Looper, Sequencer — organized around creative intent rather than technical mode. It subscribes to Dashboard's harmonic state, so it stays in sync with the rest of the session automatically.

Behavior

The interface barely changed from prototype to release. Almost all the design work happened in the behavior: how does a sound glide? How does it exit when it no longer fits?

A playing sample should never restart to follow the harmony — it should glide.

Glide curves

A pitch slide isn't one thing. Its shape carries feeling. I designed a family of glide behaviors, from mechanical to expressive.

The most interesting one is entirely Nathan Ho's idea — his research on how the human voice slides into a pitch. A singer doesn't hit a note in a straight line; they anticipate and overshoot. I implemented it.

Exit modes

What happens when a sample can't fit the new scale? Three exit behaviors — stutter decay, filter sweep, doppler drop — that can be stacked.

I've performed with it for years; the plugin port brings that to any DAW.

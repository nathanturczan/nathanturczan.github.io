2024–present MIDI Plugin

Tonalign started because I wanted the MIDI equivalent of AutoTune — my first real plugin.

At the time I was building Scale Navigator as a web application. I knew almost nothing about JUCE or C++. I posted on the JUCE forum looking for help and met Brady Boettcher. This was mid-pandemic; he was often in a completely different time zone. Over the next few months we pair-programmed the first version together — Brady brought the C++ and JUCE expertise, I brought the design and the Dashboard integration concept. Watching him work fundamentally changed how I approached audio software.

What it does

AutoTune corrects audio pitch. Tonalign does the same thing for MIDI. You play a note that doesn't fit the current scale, and it snaps to the nearest note that does.

If Dashboard is running in the same session, Tonalign subscribes to its harmonic state. Change the scale in Dashboard and every instance of Tonalign updates instantly.

Iteration

Held notes

What happens when you're holding a note and the scale changes underneath you?

There isn't a universally correct answer. A pianist expects the note to stay where it is. A synth player might expect it to glide. Someone triggering orchestral samples may want the old note cut off entirely.

Rather than pick one behavior, I shipped three modes — glide, stay, and cut — with glide as the default because it preserves harmonic continuity during live performance.

Where it is now

Tonalign ships as part of the Scale Navigator ecosystem and is the MIDI correction layer used in every live session I run.

The three held-note modes cover most playing situations. Edge cases still come up — overlapping corrections during fast passages, or scale changes that arrive mid-sustain. Those are the things I keep tuning.
